# RAG from Zero to Hero 项目指南

## 项目概述

本项目是一套完整的检索增强生成（Retrieval-Augmented Generation, RAG）技术实现方案，包含21个专题笔记本，覆盖从基础实现到高级优化的全技术栈。通过模块化设计，开发者可以系统性地掌握**RAG核心组件及其进阶技巧**，包括**多粒度分块策略（语义/命题/上下文分块）、混合检索引擎（向量+关键词融合）、动态上下文管理（压缩/扩展/重组）、智能查询理解（重写/退步/分解）、假设文档嵌入（HyDE）技术、自适应检索路由（问题类型感知）、自优化反馈循环（Self-RAG）、实时纠错机制（CRAG）、强化学习调优（RL RAG）、多模态处理（文本+图像联合检索）、知识图谱增强（Graph RAG）、分层索引优化（十亿级文档支持）等核心技术亮点**。

## 技术专题分类

### 基础组件
- ​**​数据预处理​**​  
  ▸ ​**​语义分块（Semantic Chunking）​**​：采用百分位法、标准差法和IQR法进行语义感知的文本分割，相比固定长度分块可提升15-20%的检索准确率   
  ▸ ​**​上下文分块头（Contextual Chunk Headers）​**​：为每个文本块添加文档标题/章节头信息，解决传统分块导致的上下文丢失问题  
  ▸ ​**​命题分块（Proposition Chunking）​**​：将文档分解为原子性事实陈述，实现更精确的细粒度检索
- ​**​检索优化​**​  
  ▸ ​**​混合检索（Fusion Retrieval）​**​：结合向量搜索与BM25关键词检索，通过分数归一化和加权融合提升召回率  
  ▸ ​**​上下文压缩（Contextual Compression）​**​：对检索结果进行二次过滤，保留与查询最相关的30-50%内容，降低噪声干扰  
  ▸ ​**​相关段提取（Relevant Segment Extraction）​**​：基于检索结果的聚类分析，提取连续文本段而非孤立块，提升上下文连贯性

### 高级增强
- ​**​查询优化​**​  
  ▸ ​**​查询转换（Query Transformations）​**​：实现查询重写、退步提示和子查询分解三种技术，使原始查询的意图表达更充分  
  ▸ ​**​假设文档嵌入（HyDE）​**​：通过LLM生成假设性答案文档作为查询扩展，有效解决短查询与长文档的语义鸿沟  
  ▸ ​**​自适应检索（Adaptive Retrieval）​**​：根据问题类型（事实型/分析型/观点型）动态选择最优检索策略
- ​**​架构创新​**​  
  ▸ ​**​图增强RAG（Graph RAG）​**​：构建知识图谱关系网络，支持通过节点遍历实现关联概念检索  
  ▸ ​**​分层索引（Hierarchical Indices）​**​：采用文档摘要→细节的两级检索架构，大幅降低大规模文档集的搜索耗时  
  ▸ ​**​多模态RAG（Multi-Modal RAG）​**​：集成图像captioning技术，实现对文档中图表信息的联合检索
- ​**​动态优化​**​  
  ▸ ​**​自优化RAG（Self-RAG）​**​：引入检索必要性判断、文档相关性评估等6个反射点，实现生成过程的自监督  
  ▸ ​**​纠错RAG（CRAG）​**​：当本地知识不足时自动触发网络搜索，动态校正检索结果  
  ▸ ​**​强化学习RAG（RL RAG）​**​：基于用户反馈构建奖励模型，持续优化检索和生成策略  
  ▸ ​**​反馈循环（Feedback Loop）​**​：记录成功Q&A对并调整文档权重，实现系统能力的渐进式提升

## 典型应用场景
1. ​**​知识密集型QA系统​**​  
   - 组合文档增强（问题生成）与命题分块技术，解决复杂事实型查询  
   - 采用HyDE处理"特斯拉2023年毛利率是多少？"等具象问题  
   - 通过CRAG机制保证时效性数据的准确性
2. ​**​多模态内容分析​**​  
   - 对科研论文中的实验流程图进行captioning生成  
   - 联合检索文本描述与图像特征（如"展示神经网络架构的图示"）  
   - 特别适用于医疗影像报告解析等专业领域
3. ​**​动态知识系统​**​  
   - 在客服场景中通过反馈循环持续优化FAQ库  
   - 根据对话记录自动调整分层索引的权重分布  
   - 利用自适应检索处理不同类型客户咨询
     
## 部署流程

### 创建并激活环境
```
conda create -n rag python=3.11
conda activate rag 
```
### 安装jupyter内核
```
conda install jupyter ipykernel -y
```
### 安装所需库
```
cd RAG-skill/build-rag-from-scratch
pip install -r requirements.txt
```
### 设置API-KEY
在.env文件中设置填写你的openai-api-key，设置好之后即可运行各个notebook
